Si tu proyecto IoT captura datos personales en Colombia, debes cumplir la Ley 1581 de 2012. Esta ley exige consentimiento previo, politicas de privacidad claras y la implementacion efectiva de derechos ARCO.

Idea clave: en IoT, casi todo puede volverse dato personal si permite inferir habitos o identificar personas.

Que datos se consideran personales

Un dato es personal si identifica o puede identificar a una persona. En IoT, incluso datos "ambientales" pueden ser personales por inferencia.

Obligaciones principales

1. Politica de privacidad

  • Que datos se capturan y por que.
  • Con quien se comparten.
  • Como se protegen.
# Politica de Privacidad - Termostato Inteligente

## Datos Capturamos
- Temperatura configurada por el usuario
- Horarios de uso (encendido/apagado)
- Ubicacion del dispositivo (solo casa)

## No Capturamos
- Audio, video
- Datos de otros dispositivos
- Conversaciones

## Como Protegemos
- Encriptacion TLS 1.3 en transito
- Almacenamiento cifrado (AES-256)
- Acceso limitado al equipo

## Sus Derechos (ARCO)
Puede solicitar Acceso, Rectificacion, Cancelacion u Oposicion
escribiendo a privacidad@ejemplo.com

2. Autorizacion (consentimiento)

  • Previa: antes de capturar datos.
  • Informada: el usuario sabe que entrega.
  • Explicita: opt-in, no opt-out.

Acceso (A)

// Response 200

Rectificacion (R)

// PUT /api/v1/user/data/profile
{
  "field": "email",
  "old_value": "old@email.com",
  "new_value": "new@email.com",
  "reason": "Email actualizado"
}

Cancelacion / Supresion (C)

// DELETE /api/v1/user/data
// Response 200
{
  "status": "deleted",
  "data_deleted": 1520,
  "data_retained": ["billing_records"],
  "retained_reason": "legal_requirement"
}

Oposicion (O)

// POST /api/v1/user/consent
{
  "data_type": "location",
  "purpose": "third_party_sharing",
  "consent": false
}

Seguridad minima obligatoria

  • Encriptacion en transito (TLS 1.2+).
  • Encriptacion en reposo (AES-256 equivalente).
  • Control de acceso y autenticacion.
  • Auditoria y logs de accesos.
Deber de informar: vulnerabilidades que afecten datos personales deben reportarse a la SIC en 15 dias habiles.

Checklist de cumplimiento

  • Politica de privacidad redactada y publicada.
  • Consentimiento explicito implementado.
  • API ARCO implementada (A, R, C, O).
  • Encriptacion TLS 1.3 habilitada.
  • Encriptacion at-rest configurada.
  • Logs de auditoria activados.
  • Procedimiento de retencion definido.
  • Minimizacion de datos aplicada.
