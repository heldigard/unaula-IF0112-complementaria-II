# Ley 1581 - Guía para Desarrolladores IoT

**Clase 14: Marco Legal | Unidad 2**

## Resumen Ejecutivo para Desarrolladores

Si tu proyecto IoT captura datos personales en Colombia, **debes cumplir la Ley 1581 de 2012**.

## ¿Qué datos son "personales"?

Son personales los datos que:
- Identifican o pueden identificar a una persona
- Ejemplos: nombre, ID, ubicación precisa, biométricos, IP, hábitos

## Datos Comunes en IoT que son Personales

| Tipo | Dato IoT | ¿Es personal? |
|------|----------|---------------|
| **Ubicación** | GPS del dispositivo | Sí |
| **Uso** | Horarios de encendido | Sí (patrón de vida) |
| **Ambiente** | Temperatura habitación | Sí (inferencia presencia) |
| **Biométricos** | Huella, voz | Sí (altamente sensible) |
| **Salud** | Frecuencia cardíaca | Sí (especial) |
| **Consumo** | Uso energía | Sí (inferencia hábitos) |
| **Agregado** | Promedio de zona | No (no identifica individuo) |

## Obligaciones del Desarrollador IoT

### 1. Política de Privacidad

Debe incluir:
- ¿Qué datos se capturan?
- ¿Para qué se usan?
- ¿Con quién se comparten?
- ¿Cómo se protegen?

**Ejemplo:**
```markdown
# Política de Privacidad - Termostato Inteligente

## Datos Capturamos
- Temperatura configurada por el usuario
- Horarios de uso (encendido/apagado)
- Ubicación del dispositivo (solo casa)

## No Capturamos
- Audio, video
- Datos de otros dispositivos
- Conversaciones

## Cómo Protegemos
- Encriptación TLS 1.3 en tránsito
- Almacenamiento cifrado (AES-256)
- Acceso limitado al equipo

## Sus Derechos (ARCO)
Puede solicitar Acceso, Rectificación, Cancelación u Oposición
escribiendo a privacidad@ejemplo.com
```

### 2. Autorización (Consentimiento)

**Requisitos:**
- Debe ser **previa** (antes de capturar)
- Debe ser **informada** (usuario sabe qué entrega)
- Debe ser **explícita** (opt-in, no opt-out implícito)

**Ejemplo en App:**
```
┌─────────────────────────────────────┐
│  Termostato Inteligente              │
│                                     │
│  Para funcionar, el termostato     │
│  necesita acceder a:                │
│                                     │
│  ☑ Ubicación del dispositivo       │
│  ☑ Datos de uso                     │
│                                     │
│  Acepto los Términos y Política     │
│  de Privacidad                       │
│                                     │
│  [  Rechazar  ]    [  Aceptar  ]   │
└─────────────────────────────────────┘
```

### 3. Derechos ARCO - Implementación Técnica

#### Acceso (A)

**Requisito:** El usuario puede ver qué datos tiene el sistema.

**Implementación API:**
```javascript
// GET /api/v1/user/data
// Response 200
{
  "user_id": "user_123",
  "data_collected": {
    "temperature_readings": 1520,
    "schedule": {"mon": [{"start": "07:00", "end": "23:00"}]},
    "device_location": "Medellín"
  },
  "retention_period": "90 days"
}
```

#### Rectificación (R)

**Requisito:** El usuario puede corregir datos errados.

**Implementación:**
```javascript
// PUT /api/v1/user/data/profile
{
  "field": "email",
  "old_value": "old@email.com",
  "new_value": "new@email.com",
  "reason": "Email actualizado"
}
```

#### Cancelación (C) / Supresión

**Requisito:** El usuario puede pedir borrar sus datos.

**Implementación:**
```javascript
// DELETE /api/v1/user/data
// Response 200
{
  "status": "deleted",
  "data_deleted": 1520,
  "data_retained": ["billing_records"],
  "retained_reason": "legal_requirement"
}
```

#### Oposición (O)

**Requisito:** El usuario puede negar uso específico de datos.

**Implementación:**
```javascript
// POST /api/v1/user/consent
{
  "data_type": "location",
  "purpose": "third_party_sharing",
  "consent": false
}
```

### 4. Seguridad

**Mínimo obligatorio:**
- Encriptación en tránsito (TLS 1.2+)
- Encriptación en reposo (AES-256 equivalente)
- Control de acceso (autenticación)
- Auditoría (logs de accesos)

### 5. Deber de Informar

Si hay **vulnerabilidad** que afecte datos personales:
- Informar a la SIC en 15 días hábiles
- Informar a los titulares si es grave

## Sanciones

| Infracción | Sanción Máxima |
|------------|-----------------|
| No informar política | Multa hasta 300 SMMLV |
| No obtener autorización | Multa hasta 600 SMMLV |
| No cumplir ARCO | Multa hasta 1000 SMMLV |
| Comprometer seguridad | Multa hasta 1000 SMMLV + cierre |

**1 SMMLV 2025 ≈ $260 USD**
**1000 SMMLV ≈ $260,000 USD**

## Checklist de Cumplimiento

Antes de lanzar tu proyecto IoT:

- [ ] Política de privacidad redactada y publicada
- [ ] Consentimiento explícito implementado
- [ ] API ARCO implementada (A, R, C, O)
- [ ] Encriptación TLS 1.3 habilitada
- [ ] Encriptación at-rest configurada
- [ ] Logs de auditoría activados
- [ ] Procedimiento de retención definido
- [ ] Política de minimización de datos aplicada
