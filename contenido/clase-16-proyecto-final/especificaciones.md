# Especificaciones del Proyecto Final

**Clase 16: Proyecto Final | Unidad 2**

## Objetivo General

Diseñar y prototipar un **sistema IoT completo** que resuelva un problema real, aplicando los conceptos aprendidos en el curso.

## Requisitos Mínimos

### 1. Hardware (Físico)

- [ ] Microcontrolador (ESP32 o similar)
- [ ] Mínimo 2 sensores diferentes
- [ ] Mínimo 1 actuador
- [ ] Circuito alimentación apropiado

### 2. Software (Código)

- [ ] Lectura de sensores funcionando
- [ ] Lógica de decisión implementada
- [ ] Actuador controlado según lógica
- [ ] Conectividad WiFi activa
- [ ] Comunicación con nube/servidor

### 3. Conectividad (Red)

- [ ] Protocolo de comunicación (MQTT/HTTP)
- [ ] Interfaz web o app móvil
- [ ] Visualización de datos en tiempo real

### 4. Documentación

- [ ] Diagrama de arquitectura
- [ ] Diagrama de conexiones físicas
- [ ] Código comentado
- [ ] Manual de usuario
- [ ] Video demostrativo

## Ideas de Proyectos

### Hogar Inteligente
- Control de iluminación con sensores presencia
- Sistema de riego automático para plantas
- Monitoreo de calidad del aire interior
- Cerradura inteligente con reconocimiento facial

### Agricultura
- Sistema de riego inteligente para huerta
- Invernadero automatizado (temp, humedad, luz)
- Monitoreo de cultivos con cámara y análisis
- Sistema de alerta de plagas

### Salud
- Monitor de signos vitales con alertas
- Sistema de medicación inteligente (recordatorios)
- Silla de rueda con sensores de presión
- Dispensador automático de medicamentos

### Ambiente
- Estación meteorológica local
- Sistema de alerta de inundación
- Monitor de calidad de agua (acueducto)
- Contador de consumo energético con optimización

### Seguridad
- Alarma de intrusos con cámara
- Sistema de alerta de incendios
- Caja fuerte con apertura remota
- Monitoreo de vehículos (GPS)

## Rúbrica de Evaluación

| Criterio | Peso | Excelente (4) | Bueno (3) | Aceptable (2) | Insuficiente (1) |
|----------|------|---------------|-----------|---------------|------------------|
| **Hardware** | 20% | Funciona completo, conexión limpia | Funciona con algunos bugs | Prototipo incompleto | Sin prototipo |
| **Software** | 25% | Código limpio, documentado, modular | Código funcional con comentarios básicos | Código sin comentarios | Código no funcional |
| **Documentación** | 20% | Informe completo, diagramas claros | Informe con información básica | Informe incompleto | Sin informe |
| **Innovación** | 15% | Solución creativa, no trivial | Solución conocida bien implementada | Solución básica | Copia de proyecto existente |
| **Presentación** | 10% | clara, demo en vivo, responde preguntas | Presentación clara, demo video | Presentación básica | Sin presentación |
| **Defensa** | 10% | Responde todas las preguntas con profundidad | Responde mayoría de preguntas | Responde algunas preguntas | No responde |
