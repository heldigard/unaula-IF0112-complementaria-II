¿Cómo IoT transforma empresas y ciudades?

🎯 Aprendizaje Clave de esta Sección

IoT es el habilitador fundamental de la transformación digital, permitiendo que empresas y ciudades pasen de reaccionar a anticipar, de operar con datos desactualizados a tener información en tiempo real.

¿Por qué la Transformación Digital?

Datos del Mundo Real

  • 75% de empresas han implementado o están implementando plataformas digitales (IDC 2024)
  • US$1.2 billones (trillions) en IoT para 2026

Ejemplos en Colombia

  • 🚦 Medellín: Semáforos inteligentes con sensores
  • 🏭 Grupo Nutresa: Sensores en líneas de producción
  • 🏢 EPM: Smart grid para gestión energética
Analogía: Una empresa sin IoT es ciega y sorda. Con IoT, siente y reacciona en tiempo real.

Términos Clave y Definiciones

Nota: Los siguientes términos se utilizan frecuentemente en IoT y transformación digital. Comprenderlos es fundamental para seguir el contenido de esta clase.
Término/Acrónimo Definición Ejemplo
IoT Internet of Things (Internet de las Cosas): Red de objetos físicos con sensores, software y conectividad para intercambiar datos Termostato conectado que se controla desde el celular
KPI Key Performance Indicator (Indicador Clave de Desempeño): Métrica cuantificable para evaluar el éxito de una actividad u objetivo Porcentaje de reducción de consumo energético
ROI Return on Investment (Retorno sobre Inversión): Razón entre ganancia obtenida e inversión realizada Invertir $100 y recuperar $150 en un año = 50% ROI
PLC Programmable Logic Controller (Controlador Lógico Programable): Computadora industrial para automatizar procesos de manufactura Controla una línea de empaque detectando productos defectuosos
SCADA Supervisory Control and Data Acquisition (Control Supervisorio y Adquisición de Datos): Sistema industrial para monitorear y controlar procesos a distancia Centro de control que monitorea 50 plantas remotamente
AMI Advanced Metering Infrastructure (Infraestructura de Medición Avanzada): Sistema de medidores inteligentes con comunicación bidireccional Medidor eléctrico que envía consumo cada 15 minutos
PLC (comunicación) Power Line Communication (Comunicación por Línea de Poder): Transmisión de datos por cables de energía eléctrica El medidor envía datos por el mismo cable que lleva electricidad
MQTT Message Queuing Telemetry Transport: Protocolo ligero de mensajería para dispositivos con recursos limitados Un sensor de temperatura publica datos que múltiples subscriptores reciben
TLS Transport Layer Security: Protocolo criptográfico para comunicaciones seguras en redes Cuando ves "https://" en un sitio web, usa TLS
AES-256 Advanced Encryption Standard - 256 bits: Algoritmo de encriptación simétrica de grado militar Protege datos sensibles como números de tarjetas de crédito
V2V / V2I Vehicle-to-Vehicle / Vehicle-to-Infrastructure: Comunicación entre vehículos o entre vehículos e infraestructura Un auto alerta a otros autos sobre un choque adelante
Cobots Collaborative Robots (Robots Colaborativos): Robots diseñados para trabajar de forma segura con humanos Robot que ayuda a ensamblar piezas junto a un operario
Wearables Dispositivos tecnológicos portátiles (vestibles): Electrónica que se lleva puesta en el cuerpo Smartwatch que monitorea ritmo cardíaco
Error Común: Confundir PLC (controlador) con PLC (comunicación). Son dos tecnologías diferentes: uno es hardware industrial, el otro es un protocolo de comunicación por líneas eléctricas.

Arquitectura IoT: Las 3 Capas

🎯 Aprendizaje Clave

La arquitectura de 3 capas (Percepción → Red → Aplicación) es el modelo estándar que permite que datos fluyan desde el mundo físico hasta las aplicaciones de negocio de manera organizada y escalable.

☁️ Capa 3: Aplicación

Función: Interfaz de usuario

  • Dashboards - Visualizar KPIs
  • Apps - Control remoto
  • Alertas - Notificaciones

📡 Capa 2: Red

Función: Conectividad

  • Gateway - Agregar datos
  • Edge - Procesamiento local
  • Protocolos - Transmisión segura

🔌 Capa 1: Percepción

Función: Interfaz física

  • Sensores - Capturar datos
  • Actuadores - Ejecutar acciones
  • Hardware - Interfaz mundo real

Flujo de Datos: Ejemplo en Fábrica

Paso Capa Acción Ejemplo
1 Percepción Sensor detecta Temperatura = 85°C
2 Red Transmite dato LoRaWAN → Gateway
3 Aplicación Alerta usuario "Sobrecalentamiento"
Principio: Cada capa tiene responsabilidad única

La Industria 4.0 representa la cuarta revolución industrial, donde la fusión de lo físico y lo digital (sistemas ciberfísicos) permite fábricas autónomas, personalizadas y conectadas.

Smart Cities: Ciudades Inteligentes

🎯 Aprendizaje Clave

Una Smart City integra múltiples sistemas IoT (movilidad, energía, seguridad, ambiente) para crear una infraestructura urbana coordinada que responde en tiempo real a las necesidades ciudadanas.

Definición

Una ciudad inteligente usa tecnología IoT para mejorar la eficiencia de servicios urbanos, la calidad de vida y la sostenibilidad ambiental.

Componentes de una Smart City

🚗 Movilidad

Semáforos adaptativos, parking inteligente

💡 Energía

Smart grid, medidores inteligentes

🌳 Ambiente

Calidad aire, ruido, clima

🏛️ Gobierno

Transparencia, participación

🏥 Salud

Telemedicina, monitoreo

🔒 Seguridad

Cámaras, alertas, emergencias

🏠 Edificios

Gestión eficiente de recursos

🚰 Agua

Detección fugas, calidad

📡 Conectividad

5G / fibra / LPWAN

Desafíos de Smart Cities

🎯 Aprendizaje Clave

Implementar Smart Cities requiere equilibrar innovación técnica con consideraciones sociales (privacidad, brecha digital) y viabilidad económica (ROI, sostenibilidad financiera).

Desafíos Técnicos 🔧 i

🔌 Interoperabilidad

  • Múltiples fabricantes
  • Protocolos diferentes
  • Sistemas legacy

🔒 Ciberseguridad

  • Infraestructura crítica
  • Miles de vulnerabilidades
  • Ataques sofisticados

📈 Escalabilidad

  • Crecimiento exponencial
  • Mantenimiento masivo
  • Costos operativos

Desafíos Sociales 👥 i

Desafío Descripción
👁️ Privacidad ¿Quién accede a mis datos?
🌍 Brecha Digital No todos tienen acceso igual
🤝 Aceptación Confianza en la tecnología
⚠️ Dependencia ¿Qué pasa si falla el sistema?

Desafíos Económicos 💰 i

Desafío Descripción
💵 Costo Inicial Inversión millonaria
📊 ROI Retorno a largo plazo
🔄 Sostenibilidad Modelo de negocio viable

Privacidad en Ciudades Inteligentes

✅ Más Seguridad

  • 📷 Más cámaras
  • 🔒 Menos crimen
  • 🚨 Respuesta rápida

⚠️ Menos Privacidad

  • 👁️ Vigilancia masiva
  • 🗃️ Datos ciudadanos
  • 🤖 Perfilamiento

Principios para el Equilibrio

Principio Aplicación
🔍 Transparencia Informar qué datos se recopilan
✅ Consentimiento Opt-in donde sea posible
📉 Minimización Solo datos necesarios
⏱️ Retención Limitada Borrar cuando no se necesiten
🔎 Auditorías Revisiones independientes
